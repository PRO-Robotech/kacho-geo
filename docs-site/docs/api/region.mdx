# Region

**Region** — верхнеуровневый элемент топологии платформы Kachō: географический регион, внутри
которого живут зоны доступности ([Zone](/api/zone)). Это глобальный cluster-scoped read-only
справочник: регион **не принадлежит** проекту или аккаунту, он описывает саму инфраструктуру.
Consumer-сервисы (Compute, VPC и др.) ссылаются на регион по id и валидируют его через
`RegionService.Get`.

Идентификатор региона **назначается администратором** явно (`region-1`) и **неизменяем** после
создания (immutable PK). Наполняет справочник администратор через `InternalRegionService`
(cluster-internal); публичный API только читает.

## Методы

Чтение (`Get` / `List`) — публичные и **синхронные**, на listener `:9090`. Администрирование
(`Create` / `Update` / `Delete`) — `InternalRegionService` на cluster-internal listener `:9091`,
**асинхронное**: RPC возвращает `Operation` (`done: false`), результат забирается поллингом
`OperationService.Get(id)` (см. [Operations (LRO)](/api/operations)).

### Get — получить регион

<ApiOperation method="GET" endpoint="/geo/v1/regions/{id}">

Возвращает регион по идентификатору. Публичный read (требует отношение `viewer`).

<CodeBlock language="bash">
  {dedent`
    curl 'http://localhost:18080/geo/v1/regions/region-1' \\
      -H 'Authorization: Bearer <JWT>'
  `}
</CodeBlock>

<CodeBlock language="json">
  {dedent`
    {
      "id": "region-1",
      "name": "Region 1",
      "createdAt": "2026-06-24T10:00:00Z"
    }
  `}
</CodeBlock>

</ApiOperation>

### List — список регионов

<ApiOperation method="GET" endpoint="/geo/v1/regions">

Возвращает список регионов с cursor-пагинацией (`pageSize`, `pageToken` → `nextPageToken`).
Публичный read.

<CodeBlock language="bash">
  {dedent`
    curl 'http://localhost:18080/geo/v1/regions?pageSize=50' \\
      -H 'Authorization: Bearer <JWT>'
  `}
</CodeBlock>

<CodeBlock language="json">
  {dedent`
    {
      "regions": [
        { "id": "region-1", "name": "Region 1", "createdAt": "2026-06-24T10:00:00Z" }
      ],
      "nextPageToken": ""
    }
  `}
</CodeBlock>

</ApiOperation>

### Create — создать регион (admin, internal)

<ApiOperation method="POST" endpoint="/geo/v1/regions">

Создаёт регион с **явно заданным** администратором id. Асинхронная операция: RPC возвращает
`Operation` (`done: false`), созданный `Region` приходит в `response` при `done: true`.
Доступна только на cluster-internal listener (`InternalRegionService`, требует отношение
`system_admin`).

<CodeBlock language="bash">
  {dedent`
    curl -X POST 'http://localhost:19091/geo/v1/regions' \\
      -H 'Authorization: Bearer <admin-JWT>' \\
      -H 'Content-Type: application/json' \\
      -d '{ "id": "region-2", "name": "Region 2" }'
  `}
</CodeBlock>

<CodeBlock language="json">
  {dedent`
    {
      "id": "geoXXXXXXXXXXXXXXXX",
      "description": "create region",
      "createdAt": "2026-06-24T10:10:00Z",
      "done": false,
      "metadata": {
        "@type": "type.googleapis.com/kacho.cloud.geo.v1.CreateRegionMetadata",
        "regionId": "region-2"
      }
    }
  `}
</CodeBlock>

Затем поллинг `GET /operations/{id}` до `done: true` — в `response` будет созданный `Region`
(см. [Operations (LRO)](/api/operations)).

</ApiOperation>

### Update — изменить регион (admin, internal)

<ApiOperation method="PATCH" endpoint="/geo/v1/regions/{id}">

Меняет изменяемые поля региона (на данный момент — `name`). Идентификатор изменить нельзя
(immutable PK). Асинхронная internal-операция: возвращает `Operation`, обновлённый `Region` —
в `response` при `done: true`.

<CodeBlock language="bash">
  {dedent`
    curl -X PATCH 'http://localhost:19091/geo/v1/regions/region-2' \\
      -H 'Authorization: Bearer <admin-JWT>' \\
      -H 'Content-Type: application/json' \\
      -d '{ "name": "Region 2 (renamed)" }'
  `}
</CodeBlock>

</ApiOperation>

### Delete — удалить регион (admin, internal)

<ApiOperation method="DELETE" endpoint="/geo/v1/regions/{id}">

Удаляет регион. Асинхронная internal-операция: возвращает `Operation`, при успехе `response` —
`google.protobuf.Empty`. Если к региону привязана хотя бы одна [Zone](/api/zone), удаление
блокируется внешним ключом `ON DELETE RESTRICT` — эта ошибка приходит **внутри** операции
(`error` = `FAILED_PRECONDITION` при `done: true`), сначала удаляются зоны.

<CodeBlock language="bash">
  {dedent`
    curl -X DELETE 'http://localhost:19091/geo/v1/regions/region-2' \\
      -H 'Authorization: Bearer <admin-JWT>'
  `}
</CodeBlock>

</ApiOperation>

## Ошибки и граничные случаи

<table>
  <thead><tr><th>Ситуация</th><th>Код</th><th>Где / поведение</th></tr></thead>
  <tbody>
    <tr><td>Get несуществующего региона</td><td><code>NOT_FOUND</code></td><td>sync gRPC-ошибка · <code>"Region &lt;id&gt; not found"</code></td></tr>
    <tr><td>Delete несуществующего региона</td><td><code>NOT_FOUND</code></td><td>в <code>Operation.error</code> (async)</td></tr>
    <tr><td>Create с уже занятым id</td><td><code>ALREADY_EXISTS</code></td><td>в <code>Operation.error</code> (async) · PK-конфликт по <code>id</code></td></tr>
    <tr><td><strong>Delete региона, к которому привязаны зоны</strong></td><td><code>FAILED_PRECONDITION</code></td><td>в <code>Operation.error</code> (async) · FK <code>Zone.regionId → Region.id ON DELETE RESTRICT</code>; сначала удалить зоны</td></tr>
    <tr><td>Малформ / пустой id в admin-мутации</td><td><code>INVALID_ARGUMENT</code></td><td>sync gRPC-ошибка · <code>Operation</code> не создаётся</td></tr>
    <tr><td>Попытка изменить <code>id</code></td><td>—</td><td><code>id</code> — immutable PK; задаётся только при Create, не меняется</td></tr>
    <tr><td>Read без отношения <code>viewer</code> / admin без <code>system_admin</code></td><td><code>PERMISSION_DENIED</code></td><td>sync · Authz-Check не прошёл (до записи <code>Operation</code>)</td></tr>
  </tbody>
</table>

:::warning Internal-поверхность
`Create` / `Update` / `Delete` обслуживаются `InternalRegionService` на cluster-internal
listener (`:9091`) и **не публикуются** на external endpoint. Их REST-проекция доступна только
на internal mux `api-gateway`.
:::
